## Installation Steps

### 1. Clone & install dependencies

```bash
git clone https://github.com/shipsecai/studio.git
cd studio
./install.sh
```

<Note>
  **macOS users:** The install script handles SDK path configuration automatically. If running `bun install` directly, set `export SDKROOT=$(xcrun --show-sdk-path)` first.
</Note>

### 2. Configure environment variables

```bash
cp backend/.env.example backend/.env
cp worker/.env.example worker/.env
cp frontend/.env.example frontend/.env
```

### 3. Start infrastructure

```bash
just infra-up     # Postgres, Temporal, MinIO, Loki
just status       # Health check
```

### 4. Run migrations

```bash
bun run migrate
```

### 5. Start the stack

```bash
pm2 start pm2.config.cjs
pm2 status
```

## Common Commands

### Docker

```bash
just infra-up      # Start infrastructure
just infra-down    # Stop infrastructure
just infra-logs    # View logs
```

### PM2

```bash
pm2 status                    # Check process status
pm2 logs shipsec-backend      # View backend logs
pm2 logs shipsec-worker       # View worker logs
pm2 logs shipsec-frontend     # View frontend logs
pm2 restart all               # Restart all processes
```

### One-command dev stack

```bash
bun run dev:stack        # Start everything
bun run dev:stack:stop   # Stop everything
```

## Troubleshooting

### macOS: Native module build errors

```bash
export SDKROOT=$(xcrun --show-sdk-path)
bun install
```

### Temporal namespace missing

```bash
docker compose -p shipsec --profile setup up -d
```

### PM2 process crash

1. Verify `.env` values match Docker services
2. Ensure Temporal namespace exists
3. Re-run migrations and restart PM2

### Database schema drift

```bash
bun run migrate
pm2 restart all
```

### Clean rebuild

```bash
pm2 stop all
docker compose -p shipsec down --volumes
just infra-up
bun run migrate
pm2 start pm2.config.cjs
```
